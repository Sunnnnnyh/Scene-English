# SceneEnglish 实施计划

> 面向对象：AI 开发者  
> 输入依据：`memory-bank/design-document.md`、`memory-bank/tech-stack.md`  
> 约束：每一步都要小而具体；每一步完成后必须验证；本文件只写指令，不包含代码。

---

## 执行原则

1. 严格使用微信小程序原生框架和 TypeScript。
2. 第一版不接后端、不接真实 ASR、不引入跨端框架。
3. 页面只负责展示和交互，业务逻辑放在 `services` 和 `utils`。
4. 用户数据优先使用本地缓存，并通过 service 层访问。
5. 每一步都必须先完成验证，再进入下一步。
6. 不要扩大 MVP 范围；Coming soon 场景不可进入。
7. UI 相关记录见 `memory-bank/ui-notes.md`，开发阶段先以基础 UI 跑通功能，最终视觉精修后置。
8. 开发阶段参考 Figma 文件中的 `SceneEnglish Low-fi Wireframes`，但不追求像素级还原。
9. 自动化测试与微信开发者工具人工验收并存：`services`、`utils`、数据逻辑优先自动测试；页面跳转、热区点击、录音、音频播放和真机体验使用人工验收记录。
10. 每完成一个重大功能或里程碑后，必须更新 `memory-bank/progress.md` 和 `memory-bank/architecture.md`。
11. 每完成一个 Step，先等待用户验证通过；验证通过后更新 `progress.md`，如有新增或调整文件则更新 `architecture.md`，然后建议进行本地 Git commit。
12. 每完成一个阶段或可演示节点后，建议 push 到 GitHub；不要在未验证通过时提交或推送。

## 已确认执行细则

- 小程序源码统一放在 `miniprogram/` 目录，根目录保留 `memory-bank/`、`AGENTS.md`、测试配置和项目配置。
- 项目协作说明统一使用 `AGENTS.md`，不要再把单数旧文件名作为当前协作文件引用。
- 正式写代码前先初始化 `memory-bank/architecture.md`，后续新增文件或模块时持续更新。
- MVP 开发阶段使用微信开发者工具的测试号 / 测试 AppID，后续发布时再替换正式 AppID。
- 在 `miniprogram/` 下手动创建原生微信小程序 TypeScript 基础结构，不依赖 GUI 向导或第三方脚手架。
- 第 0 阶段需要初始化 `.gitignore`，忽略依赖、构建产物、临时文件、日志和本地 IDE 文件。
- 测试文件放在根目录 `tests/`，测试配置也放在根目录。
- 静态资源放在 `miniprogram/assets/`，按 `images/`、`audio/`、`icons/` 等类型分目录。
- 场景和单词数据放在 `miniprogram/data/scenes.ts`，类型定义集中放在 `miniprogram/types/index.ts`。
- 本地缓存 key 统一使用 `sceneenglish:` 前缀，用户每次操作后立即写入本地缓存。
- 课堂场景图先使用简单占位插画 / 低保真场景图，20 个单词音频先使用占位或临时可播放资源。
- 热点点击采用透明热区覆盖法，用百分比定位和尺寸管理物品区域。
- 底部导航进入 MVP：Home / Learn / Review / Me 都存在，非核心页面轻量化。
- 练习每组默认 5 题；普通练习随机抽取，优先已学词，不足 5 个时从未学词补足；错题练习按弱项优先。
- 连续两次点错物体后，高亮正确物体，并继续进入拼写或口语步骤，同时记录物体识别错误。
- `Learned x / 20` 只由 Memory Mode 中打开单词卡触发，同一单词不重复计数。
- 单词卡展示美式音标；例句和实用表达默认只显示英文，中文翻译点击展开。
- Listen + Spell 忽略大小写和首尾空格；第一次拼写错误给轻提示，第二次显示正确答案并进入下一题。
- Listen + Speak 使用录音流程 + mock 识别结果，第一次失败允许重试，第二次显示正确发音提示并记录口语错误。
- 错题统计按单词累计错误，并按错误类型分别记录次数和掌握进度。
- 错题掌握进度按错误类型计算：同一错误类型连续答对 1 次进度 50%，连续答对 2 次进度 100%，该错误类型从弱项中移除；全部弱项完成后，该词从错题夹移除。
- 错题专项练习根据错误类型自动进入对应练习：`click` 练听音点击物体，`spelling` 练 Listen + Spell，`speaking` 练 Listen + Speak。
- `click` 类型错题专项只练“听音点击物体”，不额外进入拼写或口语步骤。
- Lecture Hall、Dormitory、Cafeteria 只展示锁定状态，点击提示 `Coming soon`，不做真实解锁。
- Me 页面采用轻量个人页，不做登录，展示昵称占位、总学习词数、收藏数、错题数，并标明当前使用 mock ASR。
- 收藏夹和错题夹属于跨场景复习资产，不放在具体场景学习首页；后续通过 Home、Review 或 Me 等全局入口进入。
- MVP 不做 streak / 连续学习天数，不做正式埋点，只保留必要的 console 调试日志。
- 每个关键页面的首次引导只出现一次，并记录到本地存储。
- 收藏夹、错题夹等页面需要基础空状态和轻量跳转操作。
- 错误反馈采用轻问题 toast、关键阻断 modal 的方式。
- 界面主体使用中文，学习内容保留英文，并提供可展开中文辅助。
- GitHub remote 已连接到 `https://github.com/Sunnnnnyh/Scene-English.git`，本地默认分支为 `main`。
- Git 节奏采用“Step 级别本地 commit，阶段级别或可演示节点 push 到 GitHub”。

- 验证手动移出后不再出现在错题列表。

完成标准：

### Step 2.9 实现抽题服务

任务：

- 生成每轮 5 题。

- 错题专项练习按弱项优先排序，优先错误次数高、掌握进度低的词。
- 一轮内尽量不重复。
- 支持单词总量不足时的兜底。

验证：

- 编写并运行抽题服务测试。
- 验证普通练习优先抽取已学词。
- 验证已学词不足 5 个时会从未学词补足。
- 验证错题专项练习优先抽取弱项。
- 验证题目数量为 5。
- 验证一轮内没有重复单词，除非数据不足。

完成标准：

- 听力默写、口语模式和错题专项练习都能使用同一抽题逻辑。
