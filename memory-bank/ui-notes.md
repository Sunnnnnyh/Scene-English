## 2. 视觉方向

整体方向：

> 浅色系、清晨蓝调、柔和日出暖色、轻快活泼但不幼稚。

关键词：

- Light blue hour learning UI
- Mist white background
- Soft sunrise accents
- Airy and playful
- Calm and friendly
- Rounded cards
- Clear navy outlines
- Gentle learning atmosphere

希望传达的感觉：

- 像一个轻松、友好的英语学习工具；
- 有一点插画和游戏感，但不是儿童 App；
- 适合有英语环境交流需求、留学准备需求的学生；
- 看起来清新、现代、有记忆点；
- 长时间使用不刺眼、不疲劳。

---

## 3. 色彩方向

主题色方向：

- 蓝：作为主要学习氛围色，偏清晨蓝、雾蓝、蓝灰，不使用深色满屏蓝。
- 橙：作为主行动色，用于 Start、OK、Practice 等关键按钮。
- 红：作为温暖点缀和错误/提醒状态，偏柔和珊瑚红，不使用刺眼正红。

推荐色彩角色：

| 色彩角色 | 用途 | 感觉 |
|---|---|---|
| Mist White | 页面主背景 | 清爽、轻盈、留白充足 |
| Blue Hour | 场景卡、学习面板、次级按钮 | 冷静、学习感、清晨蓝调 |
| Soft Lavender | Coming soon、标签、辅助状态 | 活泼但克制 |
| Warm Coral Orange | 主按钮、选中、高亮热区 | 温暖、行动感 |
| Cream Yellow | 提示卡、轻引导、鼓励信息 | 友好、柔软 |
| Pale Mint | 成功反馈、正向状态 | 放松、清晰 |
| Navy Ink | 主文字、描边 | 清楚、有质感 |

需要避免：

- 大面积深紫或深蓝背景；
- 高饱和荧光黄、荧光绿；
- 强烈撞色；
- 土味渐变；
- 纯黑大面积文字；
- 过重阴影。

---

## 4. 形态与组件风格

推荐形态：

- 大圆角卡片；
- 1.5px 左右深海军蓝描边；
- 轻微阴影；
- 宽松间距；
- 清晰按钮；
- 软几何插画元素；
- 少量手绘感或不规则感，但不要影响可读性。

组件方向：

- 场景卡：大面积展示场景图，卡片圆润，有清楚 CTA。
- Coming soon 卡：可见但不抢主场景，状态明确。
- 模式入口卡：三种模式信息清楚，避免图标和文案过多。
- 单词卡：层级最重要，英文单词、音标、中文、例句、实用表达要分区。
- 练习页：进度、题目状态、操作区要明确分离。
- 录音按钮：足够大、容易发现、状态清楚。
- 收藏/错题列表：列表卡片统一，强调可复习。

## 5. 字体与文字层级

字体方向：

- 使用现代圆润 sans-serif。
- 微信小程序开发阶段可使用系统字体，保持干净可读。
- 不使用过度卡通或装饰性字体。

层级建议：

- 页面标题：大号、加粗，清晰传达当前页面。
- 主要卡片标题：中大号、加粗。
- 正文说明：14-16px，保证手机上可读。
- 辅助说明：谨慎使用小号字，只用于 caption、状态、meta 信息。

注意：

- 所有英文表达必须在手机宽度下可读；
- 长句需要换行，不允许溢出卡片；
- 按钮文字必须居中且不挤压；
- 中文翻译可以作为次级信息，不抢英文表达层级。

## 6. 当前已知 UI 问题

当前 `SceneEnglish Low-fi Wireframes` 仍有以下不足：

- 部分点击框或卡片可能存在重叠；
- 局部间距还不够精细；
- 场景图只是低保真占位，不是最终插画；
- 热区位置需要等正式场景图确定后重新校准；
- 单词卡信息较多，后续需要验证是否拥挤；
- 听力默写页面的输入区和反馈区可能需要开发后重新排布；
- 口语页面录音按钮状态还需要补充按下、识别中、成功、失败等状态；
- 收藏夹和错题夹的空状态还未完整设计；
- 错题移出提示目前只作为结构参考；
- 底部导航是否必要，需要在开发阶段结合页面路径再判断；
- Coming soon 卡片的视觉权重可能需要降低，避免抢 Classroom 主入口。

这些问题暂时不阻塞功能开发。

## 7. 开发阶段 UI 处理原则

开发阶段目标：

> 先用基础 UI 跑通功能闭环，再回头进行视觉精修。

执行原则：

- 不要求逐像素还原 Figma；
- 优先保证页面结构、功能路径和状态反馈清楚；
- 保持浅色系、圆角卡片、柔和蓝橙红点缀；
- 按钮高度至少 44px；
- 页面主要间距尽量遵循 8pt grid；
- 所有文字必须可读；
- 任何 UI 如果影响功能验证，应优先简化；
- 不在开发早期花大量时间抠插画、动效、阴影、复杂视觉细节。

开发阶段可以接受：

- 使用占位场景图；
- 使用临时热区坐标；
- 使用基础按钮和卡片；
- 先没有复杂动效；
- 列表和反馈状态先做简洁版；
- 口语模式先使用 mock ASR 状态。

开发阶段不能接受：

- 用户不知道哪里可以点击；
- 主要 CTA 不明显；
- 单词卡信息看不清；
- 输入框和按钮太小；
- 录音入口不明显；
- 错误反馈没有说明；
- 收藏和错题状态不同步。

## 8. 后续 UI 精修清单

功能 MVP 跑通后，回到 Figma 和代码中重点精修：

1. 统一页面间距和卡片尺寸。
2. 修正所有点击框和卡片重叠问题。
3. 根据正式教室插画重新标注热区。
4. 优化场景图在不同屏幕比例下的展示方式。
5. 优化单词卡信息层级。
6. 增加单词卡实用表达中文展开状态。
7. 补充听力默写的正确、第一次错误、第二次错误状态。
8. 补充口语模式的录音中、识别中、识别成功、识别失败状态。
9. 设计收藏夹空状态。
10. 设计错题夹空状态。
11. 设计错题掌握进度条：同一错误类型答对 1 次显示 50%，连续答对 2 次显示完成并移除该弱项。
12. 降低 Coming soon 卡片视觉权重。
13. 检查底部导航是否保留。
14. 统一按钮、标签、chip、进度条样式。
15. 统一深海军蓝描边粗细。
16. 控制阴影强度，避免页面显脏。
17. 检查所有英文长句换行。
18. 检查所有移动端尺寸下的可读性和可点击性。
